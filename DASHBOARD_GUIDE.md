## Overview

This guide provides step-by-step instructions to build a comprehensive **Smart Campus Intelligence Dashboard** using data from the Campus Intelligence System. The dashboard includes:

- **6 Dashboard Pages** covering overview, resource utilization, peak hours, predictive insights, anomaly detection, and recommendations
- **10 Dimension/Fact Tables** optimized for analytics
- **15+ DAX/Calculated Field Measures** for KPIs and analytics
- **Interactive Slicers** for date, building, location, and time-based filtering

### System Requirements
- **Power BI Desktop** (recommended) or **Tableau Desktop** (10 or higher)
- **10 CSV files** in `outputs/dashboard/` folder (generated by Module 6)
- 500 MB disk space minimum

## Dashboard Architecture

### Fact & Dimension Tables (10 Total)

| Table Name | Purpose | Row Count | Key Fields |
|-----------|---------|-----------|-----------|
| **fact_electricity** | Building hourly electricity usage | 4,455 | timestamp, building, units_consumed, hour, day_of_week |

| **fact_mess** | Meal service and crowd data | 1,302 | timestamp, meal_type, number_of_students, crowd_ratio |
| **fact_attendance** | Daily student attendance | 16,745 | date, student_id, class_id, attendance_status, is_present |
| **fact_peak_hours** | Hourly peak indicators | 24 | hour, avg_wifi_users, avg_units, peak_wifi, peak_electricity |
| **fact_daily_trends** | Daily aggregated metrics | 31 | date, total_units, total_users, avg_mess_entries, attendance_rate |
| **fact_weekly_trends** | Weekly aggregated metrics | 5 | week, avg_wifi_users, avg_units, avg_mess_entries, avg_attendance |
| **fact_elec_utilization** | Building-level electricity | 6 | building, total_units |
| **fact_wifi_utilization** | Location-level WiFi | 6 | location, total_users |
| **dim_recommendations** | Smart recommendations | 27 | rec_id, category, priority, recommendation, estimated_saving |

### Data Relationships

```
Dimension Hub: fact_daily_trends[date]
    ├── fact_electricity[date] ────────────────────────┐
    ├── fact_wifi[date] ────────────────────────┐
    ├── fact_mess[date] ────────────────────────┤──> fact_daily_trends
    └── fact_attendance[date] ─────────────────┘

Peak Hours Hub: fact_peak_hours[hour]
    ├── fact_electricity[hour] ──────────────────────> fact_peak_hours
    └── fact_wifi[hour] ─────────────────────────────> fact_peak_hours

Weekly Hub: fact_weekly_trends[week]
    └── fact_daily_trends[week] ─────────────────────> fact_weekly_trends

Standalone Tables:
    └── dim_recommendations (no joins needed)
```

### Cardinality Summary
- `fact_daily_trends[date]` → **One-to-Many** with all 4 fact tables
- `fact_peak_hours[hour]` → **One-to-Many** with electricity & wifi
- `fact_weekly_trends[week]` → **One-to-Many** with daily_trends

### STEP 3: Create DAX Measures

1. Right-click any table → **New Measure**
2. Paste these measures:

#### KPI Measures

```dax
// ── Average Electricity Consumption
Avg Electricity (kWh) = AVERAGE(fact_electricity[units_consumed])

// ── Peak Mess Hour
Peak Mess Hour = 
VAR PeakHour =
    CALCULATE(
        FIRSTNONBLANK(fact_mess[hour], 1),
        TOPN(1,
            SUMMARIZE(fact_mess, fact_mess[hour],
                "AvgStudents", AVERAGE(fact_mess[number_of_students])),
            [AvgStudents], DESC)
    )
RETURN
    FORMAT(PeakHour, "00") & ":00"

// ── Max WiFi Users
Max WiFi Users = MAX(fact_wifi[number_of_users])

// ── Average Attendance %
Avg Attendance % = AVERAGE(fact_daily_trends[attendance_rate])
```

#### Calculation Measures

```dax
// ── Daily Electricity Total
Daily Electricity Total = 
CALCULATE(
    SUM(fact_electricity[units_consumed]),
    ALLEXCEPT(fact_electricity, fact_electricity[date])
)

// ── Attendance Rate %
Attendance Rate % = 
DIVIDE(
    CALCULATE(SUM(fact_attendance[is_present])),
    CALCULATE(COUNT(fact_attendance[student_id]))
) * 100

// ── Energy Share % per Building
Energy Share % = 
DIVIDE(
    SUM(fact_electricity[units_consumed]),
    CALCULATE(SUM(fact_electricity[units_consumed]), ALL(fact_electricity[building]))
) * 100

// ── WiFi Utilization %
WiFi Utilization % = 
DIVIDE(
    AVERAGE(fact_wifi[number_of_users]),
    CALCULATE(MAX(fact_wifi[number_of_users]), ALL(fact_wifi))
) * 100

// ── Is Peak Hour
Is Peak Hour = 
IF(
    MAX(fact_peak_hours[peak_wifi]) = TRUE() ||
    MAX(fact_peak_hours[peak_electricity]) = TRUE(),
    1, 0
)

// ── Anomaly Count
Anomaly Count = 
CALCULATE(
    COUNT(fact_electricity[is_anomaly]),
    fact_electricity[is_anomaly] = 1
)

// ── Est Monthly Saving (Energy)
Est Monthly Saving Units = 
SUMX(
    FILTER(dim_recommendations, dim_recommendations[category] = "Energy Saving"),
    VALUE(
        SUBSTITUTE(
            SUBSTITUTE(dim_recommendations[estimated_saving], "₹", ""),
            ",", ""
        )
    )
)

// ── Crowd Reduction %
Mess Crowd Reduction % = 
DIVIDE(
    MAX(fact_mess[number_of_students]) - AVERAGE(fact_mess[number_of_students]),
    MAX(fact_mess[number_of_students])
) * 100

// ── Weekly Attendance Trend
Weekly Attendance Trend = 
CALCULATE(
    AVERAGE(fact_weekly_trends[avg_attendance]),
    ALLEXCEPT(fact_weekly_trends, fact_weekly_trends[week])
)

// ── Peak Color
Peak Color = 
SWITCH(
    TRUE(),
    MAX(fact_electricity[peak_indicator]) = "High",   "#E74C3C",
    MAX(fact_electricity[peak_indicator]) = "Medium",  "#E67E22",
    "#2ECC71"
)
```

```
// Hour Label
STR(INT([Hour])) + ":00–" + STR(INT([Hour])+1) + ":00"

// Day Name
DATENAME('weekday', [Date])

// Week Type
IF DATEPART('weekday',[Date]) >= 6 THEN "Weekend" ELSE "Weekday" END

// Peak Indicator
IF [Units Consumed] > 20 THEN "High"
ELSEIF [Units Consumed] > 10 THEN "Medium"
ELSE "Low"
END

// Crowd Level
IF [Crowd Ratio] > 0.6 THEN "High"
ELSEIF [Crowd Ratio] > 0.3 THEN "Medium"
ELSE "Low"
END

// Attendance Rate %
AVG([Is Present]) * 100

// Energy Share %
SUM([Units Consumed]) / TOTAL(SUM([Units Consumed])) * 100
```

## Troubleshooting

### Common Issues

#### Issue 1: "File not found" error when running Module 6
**Solution:** Ensure modules 2-5 have been executed first
```bash
python module2_preprocessing.py
python module3_analytics.py
python module5_recommendations.py
python module6_dashboard_prep.py
```

#### Issue 2: Relationships not appearing in Power BI
**Solution:** 
1. Verify all CSVs are imported as tables
2. Check column names match exactly (case-sensitive in some contexts)
3. Ensure date columns are formatted as Date type, not Text

#### Issue 3: DAX formula "Expected end of statement"
**Solution:**
1. Copy formulas exactly as shown (mind the indentation)
2. Verify you're in a measure, not a calculated column
3. Check for typos in column names or table names

#### Issue 4: Slicers not filtering charts
**Solution:**
1. Verify relationships exist between tables
2. Cross-filter direction must be **Both**
3. Enable "Sync Slicers" across pages (Power BI)

#### Issue 5: Anomaly data appears empty
**Solution:**
1. Verify `is_anomaly` column exists in electricity_clean.csv
2. Check that some rows have `is_anomaly = 1`
3. Ensure filter is applied: `is_anomaly = 1`

#### Issue 6: Peak hour calculations incorrect
**Solution:**
1. Verify `peak_electricity` and `peak_wifi` columns in peak_hours.csv are boolean
2. Check DAX formula uses Boolean TRUE(), not "True" (text)
3. Ensure week calculation aligns with data date range

### Performance Tips

1. **For large datasets (>100k rows):**
   - Aggregate daily/weekly views first
   - Use DirectQuery mode in Power BI
   - Limit date ranges in slicers

2. **Dashboard responsiveness:**
   - Limit simultaneous cross-filtering
   - Pre-filter recommendations by category
   - Use separate slicers instead of multi-select

3. **Calculation optimization:**
   - Replace complex DAX with pre-calculated columns
   - Use SUMPRODUCT instead of nested IFs
   - Cache measure results when possible
